Part 5 — commonly missed

Connect to the SP MCP Server

This is the step that was buried in the original instructions. It gives Claude access to everything we've built for you. Without it, skills and program materials won't load.

Open Claude Code in a fresh terminal and run:

claude mcp add --transport http --header "Authorization: Bearer YOUR_API_KEY_HERE" --scope user sp-shared https://mcp.strategicprofits.com/mcp
Replace YOUR_API_KEY_HERE with your personal API key (sent to you separately via email).
  1. Type /exit to close Claude Code, then type claude to reopen it
  2. Type: Tell me my sp-shared status — you should see your name, email, and available content
  3. Sync your files: Sync my shared content and be 100% sure to sync everything in [Your Obsidian Vault Name]

Troubleshooting

If the status check shows an error:

  1. Double-check that you copied your API key correctly in the command above
  2. Make sure you restarted Claude Code (/exit then claude)

If skills or files aren't loading right:

  1. Open Claude Code
  2. Type: Run sp-refresh and be 100% sure to refresh everything in [Your Obsidian Vault Name]
